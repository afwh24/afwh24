# Hi, I'm Alfred 👋

I build the data pipelines and evaluation systems that sit beneath LLMs — the infrastructure layer that makes AI development actually work. Currently finishing my CS degree at NTU Singapore while figuring out what comes next in the agent and RAG space.

I spent six months at HTX building production-scale AI data pipelines from scratch. That meant processing over 5.5 million GitHub repositories, collecting 860,000+ U.S. court cases, chunking 2,000+ parliamentary bills, and running a legal reasoning evaluation framework on 8x NVIDIA H100s using Qwen3-235B via vLLM. All independently, all end-to-end.

These days I'm reading deep into AI agents and RAG systems — particularly how retrieval and reasoning can be combined to build more reliable and grounded AI applications.

## Featured Projects

These are the four projects I built during my internship at HTX. Each one was scoped, built, and delivered independently.

| Project | What it does | Scale |
|---|---|---|
| [Synthetic Legal Reasoning Pipeline](https://github.com/afwh24/legal-reasoning-eval-framework) | Generates and evaluates judicial reasoning outputs using LLM-as-a-Judge | ~500 samples, Qwen3-235B on 8x H100s |
| [U.S. Court Data Acquisition Pipeline](https://github.com/afwh24/justia-court-scraper) | Scrapes U.S. Federal District Court cases from Justia Law | 860,000+ cases |
| [GitHub Source Code Dataset Pipeline](https://github.com/afwh24/github_scraping) | Curates source code datasets for LLM training, inspired by The Stack v2 | 5.5M+ repos, 600GB, ~40B tokens |
| [Legal Document Chunking Pipeline](https://github.com/afwh24/legal-document-chunker) | Transforms parliamentary bills into semantic chunks for legal AI retrieval | 2,000+ bills |
