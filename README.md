# Hi, I'm Alfred 👋

I build the data pipelines and evaluation systems that sit beneath LLMs, the infrastructure layer that makes AI development actually work. Currently finishing my CS degree at NTU Singapore.

I spent six months at HTX building four production-scale AI data pipelines from scratch, independently and end-to-end. That meant processing over 5.5 million GitHub repositories for LLM training data, collecting 860,000+ U.S. federal court cases, chunking 2,000+ parliamentary bills for legal retrieval, and building an LLM-as-a-Judge evaluation framework that ran Qwen3-235B (the Thinking variant) on 8x NVIDIA H100s via vLLM to assess synthetic legal reasoning.

Right now I'm building a local RAG system from scratch, Chroma as the vector store, Ollama for embeddings, LangChain for orchestration, partly to learn the stack properly and partly to turn my own project notes and technical learning into something queryable.

## Featured Projects

Four projects from my internship at HTX, each scoped, built, and delivered independently.

| Project | What it does | Scale |
|:---:|:---:|:---:|
| [Synthetic Legal Reasoning Pipeline](https://github.com/afwh24/legal-reasoning-eval-framework) | Generates synthetic judicial reasoning from masked court cases and evaluates it against the real outcome using LLM-as-a-Judge | ~500 samples, Qwen3-235B-Thinking on 8x H100s |
| [U.S. Court Data Acquisition Pipeline](https://github.com/afwh24/justia-court-scraper) | Scrapes U.S. Federal District Court cases from Justia Law across all 50 states | 860,000+ cases |

| [Legal Document Chunking Pipeline](https://github.com/afwh24/legal-document-chunker) | Transforms parliamentary bills into semantic chunks for legal AI retrieval | 2,000+ bills |
